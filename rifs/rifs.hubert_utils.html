rifs.hubert_utils.kmeans.learn_kmeans(feat_dir: Path, split: str, num_rank: int, km_dir: Path, n_clusters: int, percent: float = -1, init: str = 'k-means++', max_iter: int = 100, batch_size: int = 10000, tol: float = 0.0, n_init: int = 20, reassignment_ratio: float = 0.0, max_no_improvement: int = 100) → None

Build and train the KMeans clustering model. The model is saved in “{km_dir}/model.pt” :param feat_dir: The directory that stores the feature files. :type feat_dir: Path :param split: The split of data. Options: [train, valid]. :type split: str :param num_rank: The number of ranks for multi-processing in feature extraction. :type num_rank: int :param km_dir: The directory to store the KMeans clustering model. :type km_dir: Path :param n_clusters: The number of clusters. :type n_clusters: int :param percent: The percent of data for training k-means model.

If negative, use all data for training. (Default: -1)

Parameters:

  • init (str, optional) – Method for initialization. Options: [k-means++, random]. (Default: k-means++)

  • max_iter (int, optional) – Maximum number of iterations over the complete dataset. (Default: 100)

  • batch_size (int, optional) – Batch size for training the KMeans clustering model. (Default: 10000)

  • tol (float, optional) – Control early stopping based on the relative center changes as measured by a smoothed, variance-normalized of the mean center squared position changes. (Default: 0.0)

  • n_init (int, optional) – Number of random initializations that are tried. (Default: 20)

  • reassignment_ratio (float, optional) – Control the fraction of the maximum number of counts for a center to be reassigned. A higher value means that low count centers are more easily reassigned. (Default: 0.0)

  • max_no_improvement (int, optional) – Control early stopping based on the consecutive number of mini batches that does not yield an improvement on the smoothed inertia. (Default: 100)

Returns:

None
